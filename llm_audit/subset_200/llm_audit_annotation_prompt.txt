You are an annotation assistant for a Turkish token-level computer command understanding dataset.

Task:
For each JSONL input record, assign two label sequences:
1. boundary_labels
2. task_token_labels

Input fields:
- id
- domain
- raw_command
- tokens

Output format:
Return JSONL only.
Each line must be one JSON object with exactly these fields:
- id
- boundary_labels
- task_token_labels
- note

Do not add explanations, markdown, bullet points, or code fences.

Allowed boundary labels:
O, B-TASK, I-TASK

Boundary label definitions:
- B-TASK: first token of an executable task segment
- I-TASK: continuation token of the same task segment
- O: token outside executable task segments

Allowed task_token_labels:
O, ARA, ARTIR, AZALT, AÇ, BOŞLUK, BUL, BÜYÜT, DEĞİŞTİR, DİĞER, ENTER, GEÇ, GÖNDER, GİT, KAPAT, KAYDET, KAYDIR, KES, KOPYALA, KÜÇÜLT, SEKME_DEĞİŞTİR, SEÇ, SİL, TAŞI, TIKLA, YAPIŞTIR, YAZ, YAZDIR, YENİDEN_ADLANDIR, YENİ_OLUŞTUR, YÜKLE, YİNELE, İNDİR

Task-token label rules:
- If a token is not part of an executable task, use O.
- If a token is part of an executable task, use the task label for the whole task segment.
- The number of boundary_labels must be exactly equal to the number of tokens.
- The number of task_token_labels must be exactly equal to the number of tokens.
- If task_token_label is O, boundary_label should usually be O.
- If task_token_label is not O, boundary_label should be B-TASK or I-TASK.
- Negated or cancelled actions should not be labeled as executable tasks.
  Example: "dosyayı silme sadece klasöre taşı"
  "dosyayı silme" should be O, while "klasöre taşı" should be TAŞI.
- Conversational fillers such as "şey", "ya", "hayır", "yanlış oldu", "sadece", "ama" are usually O unless they are necessary parts of the executable task.
- Do not infer tasks that are not explicitly requested.
- Preserve the input id exactly.

Example input:
{"id":"EX-0001","domain":"temel_bilgisayar","raw_command":"dosyayı silme sadece yedek klasörüne taşı","tokens":["dosyayı","silme","sadece","yedek","klasörüne","taşı"]}

Example output:
{"id":"EX-0001","boundary_labels":["O","O","O","B-TASK","I-TASK","I-TASK"],"task_token_labels":["O","O","O","TAŞI","TAŞI","TAŞI"],"note":"Negated delete action ignored; move action labeled."}

Now annotate the JSONL records I provide.
